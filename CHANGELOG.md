# 更新记录

本文件用于记录每一次系统更新、优化和修复内容。以后每次上线前都应新增一条记录，并在 Git 中创建对应提交。

## 2026-06-17 - 管理端 UI 全面优化

- 新增：统一的页面容器、工具栏、指标卡片、金额显示、类型标签和表格分页组件
- 新增：admin 账号支持固定初始密码 `admin` 登录，同时保留修改后的密码登录能力
- 新增：正式域名 `mbfthb.cloud` 与 `www.mbfthb.cloud` 的服务器访问配置
- 新增：流水管理支持按筛选周期展示 DM 已发工资，并通过“实际收入 - DM已发工资 - 其他扣减”计算真实利润
- 优化：首页、登录页、DM管理、本单管理、开本记录、工资计算、报销管理、流水管理的整体视觉层级和可读性
- 优化：工资计算阶梯规则会把同月已结算车次计入阶梯起点，但不会重复计算已结算场次工资
- 优化：血染钟楼车次计入工资阶梯总车次和里程碑判断，但仍按血染工资单独计薪，避免重复叠加普通阶梯工资
- 修复：历史已结算车次显示改为按结算总车次统计，兼容旧结算中血染未计入阶梯车次的记录
- 优化：管理后台布局升级为深色侧边导航、固定顶部状态栏和更清晰的经营数据区域
- 优化：高频表格增加横向滚动、固定操作列、紧凑按钮和记录数量提示，降低误操作与等待感
- 优化：流水、报销、工资等金额字段统一格式，关键收入/支出数据更容易快速识别
- 修复：工资计算手动调整时补充血染工资字段，避免局部编辑后合计遗漏
- 修复：部署脚本生成 Nginx 配置时正确保留反向代理变量，避免部署末尾因变量展开失败
- 验证：前端生产构建通过，后端认证、首页工资日期范围、工资阶梯延续、血染阶梯计数、历史结算总车次、流水利润汇总测试通过，本地浏览器完成登录、首页、流水、工资、报销、DM、本单、开本页面巡检
- 影响范围：前端展示层、登录认证、工资计算、流水汇总利润计算和部署脚本，不改变数据库结构
- 回退方式：使用 Git 回退到本次更新前的提交，重新运行部署脚本即可恢复
